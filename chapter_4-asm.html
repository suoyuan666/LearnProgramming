学习编程

汇编介绍

C语言及大多数语言实现函数调用的机制都使用了栈数据结构提供的后入先出这一特性。函数P调用函数Q时，P及其调用P的调用链的函数都被挂起。Q运行时需要为局部变量分配新的存储空间，Q返回时需要释放那些空间。

当x86_64函数需要的存储空间超过寄存器能够存放的大小时，就会在栈上分配空间，这个过程称为它的栈帧。通过寄存器，函数P最多传递6个整数值，如果Q需要更多的参数，P可以在调用Q之前在自己的栈帧里存储好这些参数。

函数P转移到函数Q只需要简单地把程序计数器（PC）设置为Q的代码的起始位置，并记录下调用Q函数指令的下一条指令的位置，这就是call指令的作用。

call指令会把地址A压入栈中并把PC设置为Q的起始地址，压入栈中的A地址就是返回地址，即紧跟在call指令下面那条指令的地址。对应的ret指令会从栈中弹出地址A，并把PC设置为A。

数据传送

函数调用中可以通过寄存器最多传递6个整型参数——RDI、RSI、RDX、RCX、R8、R9

更多关于函数调用传参的约定可见：x86 calling conventions。

void proc(long a1, long *alp, int a2, int *a2p, short a3, short *a3p, char a4, char *a4p)
{
	*alp += a1;
	*a2p += a2;
	*a3p += a3;
	*a4p += a4;
}

;arguments passed as follows:
;	a1   in %rdi    (64 bytes)
;	a1p  in %rsi    (64 bytes)
;	a2   in %edx    (32 bytes)
;	a2p  in %rcx    (64 bytes)
;	a3   in %r8w    (16 bytes)
;	a3p  in %r9     (64 bytes)
;   a4   in %rsp+8	( 8 bytes)
;   a4p  in %esp+16 (64 bytes)
proc:
	movq	16(%rsp), %rax
	addq	%rdi, (%rsi)
	addl	%edx, (%rcx)
	addw	%r8w, (%r9)
	movl	8(%rsp), %eax
	addb	%dl, (%rax)
	ret

栈上的局部存储

有时候局部数据需要存放在内存中，例如：

  • 寄存器不足够存放所有的本地数据
  • 对一个局部变量使用取地址符&，因此需要为它产生一个地址
  • 某些局部变量是数组或结构，因此必须能够通过数组或结构引用被访问

一般来说，函数通过减少栈指针分配空间，分配的结果作为栈帧的一部分，标号为局部变量。

long swap_add(long *xp, long *yp)
{
	long x = *xp;
	long y = *yp;
	*xp = y;
	*yp = x;
	return x+y;
}
long caller
{
	long argv_1 = 534;
	long argv_2 = 1057;
	long sum = swap_add(&argv_1, &argv_2);
	long diff = argv_1 - argv_2;
	return sum * diff;
}

caller:
	subq	$16, %rsp
	movq	$534, (%rsp)
	movq	$1057, 8(%rsp)
	leaq	8(%rsp), %rsi
	movq	%rsp, %rdi
	call	swap_add
	movq	(%rsp), %rdx
	subq	8(%rsp), %rdx
	imulq	%rdx, %rax
	addq	$16, %rsp
	ret

寄存器中的局部存储空间

根据惯例，寄存器%rdx、%rbp和%r12~%r15被划分为被调用者保存寄存器。函数P调用函数Q的时候，Q必须保存这些寄存器的值，保证它们在Q返回到P与Q被调用时时一样的。为了实现这个，要么Q根本不去改变它，要么就需要先将原数据压入栈中，返回前从栈中弹出旧值。

除了上述寄存器和%rsp栈指针之外都是调用者保存寄存器，也就意味着谁都可以去修改。

long P(long x, long y)
{
	long u = Q(y);
	long v = Q(x);
	return u + v;
}

P:
	pushq	%rbp
	pushq	%rbx
	subq	%8, %rsp
	movq	%rdi, %rbp
	movq	%rsi, %rdi
	call	Q
	movq	%rax, %rbx
	movq	%rbp, %rdi
	call	Q
	addq	%rbx, %rax
	popq	%rbx
	popq	%rbp
	ret

这里push和pop的顺序相反，也体现出了栈后入先出的特性。

递归函数

long refact(long n)
{
	long result;
	if(n <= 1)
		result = 1;
	else
		result = n * refact(n-1);
	return result;
}

;n in %rdi
refact:
	pushq  %rdx
	movq   %rdi, %rbx
	movl   $1, %eax
	cmpq   $1, %rdi
	jle    .L35
	leaq   -1(%rdi), %rdi
	call   refact
	imulq  %rbx, %rax
.L35:
	popq   %rbx
	ret

从上面的示例代码中也可以看出，递归调用一个函数本身和调用其他函数没有差别。栈让函数每次调用都可以有它自己私有的状态信息（保存的返回位置和被调用者保存寄存器的值）存储空间。
