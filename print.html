数据的表示

引言

在一切开始之前，先以打印输出success的程序为例起一个头

// filename: hello.c
#include<stdio.h>
int main(void){
    printf("success\n");
    return 0;
}

~/CodeProject/C/LP
> gcc hello.c -o hello

~/CodeProject/C/LP
> ./hello

编译好后运行该程序，你会发现终端打印了一个单词——success。下面将会单独说一说这几行代码的含义。

第一行以//开头的是注释，注释是给人类阅读的，而非编译器。编译器不会管注释的内容，只会无情的删掉。

第二行以#开头的语句叫做预处理指令，它会在编译程序的整个流程中最开始的时候就被处理。#include表明该程序引入了头文件stdio.h，这个头文件中的一部分代码就描述了第三行的这条语句如何在计算机中实现的，即这条语句如何能够打印这个单词。像打印一些话这样的操作是十分基本常见的，C标准库就实现了很多基本的操作，程序员只需要引入标准头文件就可以使用已经实现好的一些功能（如上面那个例子的打印success）。

第三行定义了一个函数。函数这个概念和数学中的函数类似，有一个x，也相应的有个y。其中，int描述了y的类型（整数，小数还是字符啥的），main是这个函数的名称，括号内部的就是x的类型了，void表明没有x，而且这里函数的x可以不止有一个，可以有多个x，但是只有一个y。main这个名称也不是瞎起的，main在规定中就是程序的入口，虽然这个说法并不严谨，但可以先这么认为。一对大括号圈了一个代码块作为函数的主体，这里面就是main函数所要做的事了。有的语言可以没有入口函数，比如Python，直接从第一行开始走就行。

第四行printf就是一个函数，它接受一个x（就是success\n），但是\n并没有打印出来，这叫做转义字符，\n表示的是换行。传递过去的x通过括号包上，分号表示一个语句的结束，所以它不是将一行看作一条语句，将多条语句写在一行里也是可以的。有的语言并不是这样，比如Python没有分号，一行就表示一条语句。如果你使用Linux系统，那么在终端输入man 3 printf命令可以看到printf()函数更多的信息，如果是Windows的话，man-pages在互联网上可以在线查询，在printf(3) — Linux manual page也可以看到。

第五行return语句表明要返回一个值，return 0表示程序正常退出。int表示返回的y是个整数，这里当然return 0。事实上如果不是int而是void，这个main函数就没必要写return语句了，不过其实这个语句可以省略了，编译器会默认给它整一条这个语句），但是我并不认为这样写法的好处在哪里，如果说是为了省一行return的话，现在int类型的话也可以省这一行了。

这里的gcc命令用于编译程序，-o后面是编译生成的程序的名称，如果没有这个选项，就生成默认的a.out

这里介绍一个编译程序更好的办法——编写Makefile

Makefile可以帮助你更好地编译程序，它可以自动化的完成编译工作。写完之后，只需要一个make命令就可以完成编译，甚至根据你的需求还可以实现其他功能。如果是大型项目，一个makefile自然能省很多事。

对于上面输出success的代码，mkaefile可以这样些：

source = hello.c
target = hello

hello: $(source)
	gcc $(source) -o $(target)
	@./$(target)

clean:
	rm $(target)

~/CodeProject/C/LP
> make
gcc hello.c -o hello
success

这里只需要一个make命令就可以完成编译和运行两件事。

前两行定义两个变量，一个是需要编译的源文件，一个是编译后的目标文件。

第四行表示的意思是hello这个文件依赖于$(source)，下两行是产生这个文件具体要执行的命令。至于为什么第二条命令前会有一个@，这表示将这条命令隐藏起来不输出在终端上。

clean表示清除操作，就像clean下一行的命令那样，make clean命令就会执行rm操作。

数组

数组即一组数据，这些数据有着统一的数据类型。若要声明一个int类型的数组：

int a[5];

中括号里面的表示这个数组的元素个数，如果你要一一为它们赋初值的话：

int a[3] ={2, 3, 5};

数组是可以嵌套的（数组的元素也是一个数组），也就是所谓的多维数组，这里展示一手二维数组：

int a[2][3] = {{1, 2, 3}, {4, 5, 6}};

typedef int b [3];
b a [2] = {{1, 2, 3}, {4, 5, 6}};

第一行和后两行是等效的。

typedef关键字用于定义一个别名。

访问数组有以下两种方式，以下都以一维数组为例

int a[3] = {1, 2, 3};
printf("%d %d %d", a[0], a[1], a[2]);

这是一种访问方式，a[0]中的0表示数组下标，下标从0开始表示第一个元素。

int a[3] = {1, 2, 3};
printf("%d %d %d", *a, *a+1, *a+2);

上边是指针解引用，数组和指针在底层都是差不多的东西，如果没有数组下标，那么也都是一群地址，直接解引用就可以访问。

结构体有一个内存对齐的问题，内存对齐可以提高访问内存的效率。比如下面的代码：

#include<stdio.h>
int main(void){
    struct a{
        int h;
        double s;
        char n;
    };
    struct b{
        char n;
        int h;
        double s;
    };
    printf("sizeof(a): %d\n", sizeof(struct a));
    printf("sizeof(b): %d\n", sizeof(struct b));
    return 0;
}

AT&T与Intel汇编代码格式

这里的表述都遵循AT&T格式，ATT是GCC、objdump等工具的默认格式，Microsoft的工具和Intel的文档都是Intel格式的。两种代码有很多不同之处，GCC编译时可以通过-masm=intel来指定生成Intel格式的代码 Intel与AT&T的区别：

  • Intel代码省略了只是大小的后缀，比如push和mov，ATT则可以是pushq和movq
  • Intel省略了寄存器名字前面的“%”符号，用的是rax而不是%rax
  • Intel用不同的方式描述内存中的位置，例如“QWORD PTR [rbx]”而不是“(%rbx)”
  • 在带有多操作数的指令中操作数的顺序问题，二者是相反的。

数据传送指令

mov类

mov指令就是把数据从源位置复制到目的位置，mov类包括movb、movw、movl、movq和movabsq五个指令，区别在于操作的数据大小不同，分别是1、2、4、8字节，movabsq也是八字节。

mov S, D ;D<-S

x86_64加了一条限制：传送指令的两个操作数不能同时指向内存，这种功能的实现只能由两条指令来实现——先把数据复制到寄存器中。

大多数情况下，mov指令只会更新目的操作数指定的寄存器字节或内存位置，但是movl指令以寄存器作为目的的时候，它会把寄存器的高4位字节设置为0。这是因为x86-64采用的惯例——任何为寄存器生成32位值的 指令都会把该寄存器的高位设置为0。

常规的movq只能表示为32位补码数字的立即数为源操作数，然后把这个值符号扩展得到64位的值，放到目的位置。movabsq能够以任意的64位立即数作为源操作数，并且只能以寄存器作为目的。

movz类

movz用于将较小的值复制到较大的目的时使用，包括movzbw、movzbl、movzbq、movzwl和movzwq五个指令，每条指令最后两个字符是大小指示符，第一个字符指定了源的大小，第二个指明目的的大小。

movs类

movs用于将较小的值复制到较大的目的时使用，包括movsbw、movsbl、movsbq、movswl和movswq五个指令，每条指令最后两个字符是大小指示符，第一个字符指定了源的大小，第二个指明目的的大小。

movz和movs的异同

movz和movq都用于将数据从寄存器或内存复制到目的寄存器

movz类的指令会把目的剩余的字节填充为0，而movs类的指令通过符号扩展填充

符号扩展就是把剩余的字节设为全1或全0，例如：

movabsq $0x0011223344556677, %rax ;%rax = 0011223344556677
movb $0xAA %dl ;%dl = AA
movb %dl, %al ;%rax = 00112233445566AA
movsbq %dl, %rax ;%rax = FFFFFFFFFFFFFFAA
movzbq %dl, %rax ;%rax = 00000000000000AA

可以看到，由于十六进制A二进制表示是1010，符号扩展就会把高位字节都设置为二进制的1，也就是都设置为F

压入和弹出栈数据

栈是一种数据结构，遵循后入先出的原则，通过push指令把数据压入栈中，通过pop指令把数据弹出栈。栈可以看作一个数组，总是从数组的一端压入和弹出元素，这一端称为栈顶，x86-64中，程序栈存放在内存的某个区域。如下图所示，栈向下增长，栈顶元素的地址是所有栈中元素地址最低的，栈指针%rsp保存着栈顶元素的地址。 pushq指令的功能是把数据压入栈，将一个八字节的值压入栈中，首先要将栈指针减8，然后将值写到新的栈顶地址，所以pushq %rbp等价于下边两条指令：

sub $8, %rsp
mov %rbp, (%rsp)

上图前两栏是当%rsp为0x108,%rax为0x123时，执行指令pushq %rax的效果第三栏说明的是执行完pushq后立即执行指令popq %rdx的效果，先从内存中读出0x123，再写入寄存器%rdx中，然后%rsp值增加会0x108。但是如图所示，0x123仍然存在于内存位置0x100中，直到被覆盖。

移位运算是sal、shl、sar和shr，前两个都是左移，sar是算数右移，shr是逻辑右移。

加载有效地址是leap。

加载有效地址

加载有效地址指令leap实际上就是movq指令的变形，它的指令形式是从内存读取数据到寄存器，但实际上它根本没有引用内存。它的第一个操作数看上去是一个内存引用，实际上并不是从指定的位置读入数据，而是将地址写入目的操作数。

这条指令可以为后面的内存引用产生指针，可以简洁地描述普通算术操作，例如%rdx的值是x，那么leaq 7(%rdx, %rdx, 4), %rax就是设置寄存器%rax的值为5x+7。

long scale(long x, long y, long z)
{
	long t = x + 4 * y + 12 * z;
	return t;
}

编译时，这个函数的算术运算由三条leaq指令实现：

;long scale(long x, long y, long z)
;x in %rdi, y in %rsi, z in %rdx
scale:
	leaq (%rdi, %rsi, 4), %rax ;x+4*y
	leaq (%rdx, %rdx, 2), %rdx ;3*z
	leaq (%rax, %rdx, 4), %rax ;x+4*y +4*3z = x + 4 * y + 12 * z
	ret

一元操作和二元操作

一元操作这些指令只有一个操作数，既是源操作数也是目的操作数

二元操作的第二个操作数既是源操作数也是目的操作数，例如

sub %rcx, %rax

这个指令的含义就是%rax = %rax - %rcx，其他的同理

移位操作

移位操作第一个是移位量，第二个是要移位的数据。移位量可以是一个立即数，或者放在单字节寄存器%cl中。

左移指令有两个，sal和shl，二者效果相同，都是将右边填0。右移则分为sar（算数移位）和shr（逻辑移位）

控制

条件码

除了整数寄存器，CPU维护着一组单个位的条件码寄存器，它们描述了最近的算术或逻辑操作的属性。常用的状态码有：

  • CF：进位标志，最近操作使得最高位产生了进位，可以用来检查无符号操作的溢出
  • ZF：零标志，最近的操作得出的结果为0
  • SF：符号标志，最近的操作得到的结果为负数
  • OF：溢出标志，最近的操作导致一个补码溢出——正溢出或负溢出

leaq指令不改变任何条件码，它只用来进行地址计算。除此之外，上述指令都会设置条件码。对于逻辑操作，例如xor的进位标志和溢出标志会被设置为0，对于移位操作，进位标志将设置为最后一个被移出的位。inc和dec指令会设置溢出和零标志，但不会改变进位标志。 除了上述指令还有两种指令只设置条件码不改变任何寄存器——cmp和test指令。

cmp类有四种指令，分别对应四中数据格式。cmp指令的行为和sub保持一致。如果两个值相等，零标志位会被设置为1，其他标志可以用来确定两个操作数之间的大小关系。

test类指令和and指令的行为一致。

switch

if的判断有一个缺点，如果要进行大量的判断，它会挨个从头判断到成功的地方。switch一定程度上可以避免这个问题，它可以生成一个分支跳转表，可以直接跳转。

switch的基本格式如下：

switch (expression)
{
case /* constant-expression */:
    /* code */
    break;
default:
    break;
}

switch括号里面的是要判断的表达式，case后面（/* constant-expression */这个地方）放表达式的值，code处放具体的执行逻辑。可以有多个case，每个case的code的结尾都要有一个break，否则执行完这个case不会直接跳出switch-case，而是直接向下执行。如果表达式的值没有对应的case，那么就会走default。

这里对比两组代码的汇编代码：

//filename: if.c
#include<stdio.h>
int main(void){
    int a = 5;
    if(a == 1){
        a += 4;
    }else if(a == 2){
        a += 3;
    }else if(a == 3){
        a += 2;
    }
    return 0;
}

//filename: switch.c
#include<stdio.h>
int main(void){
    int a = 5;
    switch(a){
        case 1:
            a += 4;
            break;
        case 2:
            a += 3;
            break;
        case 3:
            a += 2;
            break;
        default:
            a += 1;
            break;
    }
    return 0;
}

循环

循环语法总共有三种实现——while循环，do-while循环和for循环。其中最常用的应该是for循环了。

for

一个典型的for循环如下：

for(int i = 0; i < 5; i++){
    <code>
}

上一段代码的含义是，先int i = 0，如果i < 5，就执行内部的code，执行完后再执行i++，然后再做判断。

for循环括号内的三个语句用;分隔，int i = 0和i++这俩部分可以省略，但是分号需要保留。

如果for循环内部的code只有一行的话，也可以像if那样不写{}。

for(int i = 0; i < 5; i++)
    <code>

while

while循环没有for循环这么花哨：

while(<condition>){
    <code>
}

while循环也可以在code只有一行的时候去掉{}。

do-while循环就是在while判断之前先执行一次。

do{
    <code>

函数调用

函数是对一堆的封装，编写程序很有可能就会遇到一些基本普偏使用的功能，比如打印字符串，这是C语言标准库里就实现好了的函数，直接引入相关头文件再调用就好；还有一些对一些程序来说没那么基本的功能，这就需要自己来实现了。

函数的使用自然也可以减少程序源代码的行数，增加程序的可读性，也可以让使用者不必在乎函数功能内部的实现逻辑直接拿来就用。

创建函数

创建函数的基本格式是

<return value type> <function name> (arguments ...){
    <code>
}

就像这样：

int add(int a, int b){
    return a + n;
}

这段代码就是创建了一个add()函数用于加两个数。

C语言中，如要调用一个函数，如果调用函数在被调用函数之前，虽然还是可以调用的，编译器会报一个Warnning。调用函数中有过有一个变量接受调用函数的返回值，如果类型不对的话，编译器会报error（至少GCC编译器是这样）。但还是推荐先在源代码的前面写上函数声明，这样一个Warnning都没有了。以add()函数为例，函数声明这么写：

int add(int a, int b);

函数声明就是告诉编译器有这么一个函数，不需要具体的函数实现。

比如下面这个有函数声明的例子：

#include<stdio.h>

int add(int a, int b);

int main(void){
    int a = 1, b = 2;
    printf("a + b = %d", add(a, b));
    return 0;
}
int add(int a, int b){
    return a + b;
}

返回值的数据类型就是在第二章中介绍的数据类型了。

汇编介绍

C语言及大多数语言实现函数调用的机制都使用了栈数据结构提供的后入先出这一特性。函数P调用函数Q时，P及其调用P的调用链的函数都被挂起。Q运行时需要为局部变量分配新的存储空间，Q返回时需要释放那些空间。

当x86_64函数需要的存储空间超过寄存器能够存放的大小时，就会在栈上分配空间，这个过程称为它的栈帧。通过寄存器，函数P最多传递6个整数值，如果Q需要更多的参数，P可以在调用Q之前在自己的栈帧里存储好这些参数。

函数P转移到函数Q只需要简单地把程序计数器（PC）设置为Q的代码的起始位置，并记录下调用Q函数指令的下一条指令的位置，这就是call指令的作用。

call指令会把地址A压入栈中并把PC设置为Q的起始地址，压入栈中的A地址就是返回地址，即紧跟在call指令下面那条指令的地址。对应的ret指令会从栈中弹出地址A，并把PC设置为A。

数据传送

函数调用中可以通过寄存器最多传递6个整型参数——RDI、RSI、RDX、RCX、R8、R9

更多关于函数调用传参的约定可见：x86 calling conventions。

栈上的局部存储

有时候局部数据需要存放在内存中，例如：

  • 寄存器不足够存放所有的本地数据
  • 对一个局部变量使用取地址符&，因此需要为它产生一个地址
  • 某些局部变量是数组或结构，因此必须能够通过数组或结构引用被访问

一般来说，函数通过减少栈指针分配空间，分配的结果作为栈帧的一部分，标号为局部变量。

寄存器中的局部存储空间

根据惯例，寄存器%rdx、%rbp和%r12~%r15被划分为被调用者保存寄存器。函数P调用函数Q的时候，Q必须保存这些寄存器的值，保证它们在Q返回到P与Q被调用时时一样的。为了实现这个，要么Q根本不去改变它，要么就需要先将原数据压入栈中，返回前从栈中弹出旧值。

除了上述寄存器和%rsp栈指针之外都是调用者保存寄存器，也就意味着谁都可以去修改。

long P(long x, long y)
{
	long u = Q(y);
	long v = Q(x);
	return u + v;
}

P:
	pushq	%rbp
	pushq	%rbx
	subq	%8, %rsp
	movq	%rdi, %rbp
	movq	%rsi, %rdi
	call	Q
	movq	%rax, %rbx
	movq	%rbp, %rdi
	call	Q
	addq	%rbx, %rax
	popq	%rbx
	popq	%rbp
	ret

这里push和pop的顺序相反，也体现出了栈后入先出的特性。
